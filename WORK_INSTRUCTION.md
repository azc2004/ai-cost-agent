## 1. 목적

운영 중인 4개 AI 서비스가 **정의된 기준 산출물 1건을 만들 때 실제로 어떤 모델을 몇 번 호출하는지**를 코드 기반으로 고정하고, 이를 바탕으로

- **환율(USD→KRW) 변동**
- **기준 수량(처리 건수) 변동**
- **모델 단가·토큰 사용량 변동**

에 따른 단가·총비용을 한 화면에서 시뮬레이션할 수 있는 Streamlit 앱을 만든다.

> 단가·토큰·환율은 시시각각 변하므로 **모두 사용자 입력(슬라이더/숫자 입력) 기반**으로 두고, 코드에 하드코딩하지 않는다. 앱은 "산출 엔진"이지 단가의 진위를 보증하는 곳이 아니다.

## 4. 비용 산출 모델 (3가지 과금 방식)

모든 비용은 아래 3가지 과금 방식 중 하나로 환산된다.

### 4.1 토큰 과금 (LLM)
```
비용(USD) = (입력토큰 × 입력단가 + 출력토큰 × 출력단가) ÷ 1,000,000
```
- 대상: gpt-4o, gpt-4o-mini, gemini-2.5-flash
- 단가: 모델별 input/output USD per 1M tokens

### 4.2 이미지 생성 과금
```
비용(USD) = 이미지 장수 × 장당단가(USD)
```
- 대상: gemini-3.1-flash-image
- 단가: 이미지 1장당 USD

### 4.3 비디오 생성 과금
```
비용(USD) = 비디오 길이(초) × 초당단가(USD)
        또는 = 영상 1개당 단가(USD)
```
- 대상: Veo 3.1 (초당 과금 가정, 사용자 선택으로 "개당 단가" 전환 가능)
- 단가: 1초당 USD

### 환율/수량 적용
```
서비스 단위당 KRW = Σ(모든 단계 비용 USD) × 환율(KRW/USD)
총비용 = 단위당 KRW × 기준 수량(건수)
```

## 6. Streamlit 화면 구성 요구사능

단일 파일 `app.py` 구조를 우선(ponytail: 최소 파일). 화면은 3영역.

### 영역 A — 전역 파라미터 (사이드바)
- 환율 슬라이더 (1,000 ~ 1,600 KRW/USD)
- 단가 테이블 편집 (`st.data_editor`): 모델별 input/output/장당/초당 단가
- 입력 토큰 기본 추정치 (서비스별 — 코드에 명시값 없으므로 사용자 조정)
- 재시도 가중률/재시도 횟수 (coordi, review)

### 영역 B — 서비스별 시나리오 (메인 상단)
4개 서비스 각각:
- 기준 수량 입력(슬라이더, 예: 1~10,000건)
- 서비스 내부 옵션:
  - coordi: 모드 선택(1/2/3-추천만/3-착장), 착장 횟수 N, 재시도 가중률
  - review: 모델 선택(gpt-4o/gpt-4o-mini), 검증 재시도 횟수
  - search: 캐시 적중률(%)
  - vod: 평균 분석 이미지 수(1~14), 비디오 길이(초), Luma fallback 확률
- 결과: **기준 산출물 1건당 KRW** + **총비용 KRW**

### 영역 C — 통합 비교표 + 차트 (메인 하단)
- 4개 서비스 단위당/총비용 비교표 (`st.dataframe`)
- 건수별 비용 증가 라인 차트 (수량 1~N 스윕) — `st.line_chart` 또는 Plotly
- 환율 민감도: 환율 변화에 따른 총비용 표

> 차트 색상/디자인은 `dataviz` 스킬 가이드라인(브랜드 중립 기본 팔레트, 라이트/다크 일관성)을 따른다.

## 8. 비고 / 보류사항

1. **입력 토큰 불확실성**: 코드에 명시된 값은 `max_tokens`(출력 상한)뿐. 입력 토큰은 서비스별 추정 기본값을 두되, UI에서 "입력 토큰 추정치"로 명시 표기(추정임을 숨기지 않는다).
2. **단가 변동**: 모든 단가는 기본값일 뿐. README에 공식 가격 페이지 링크/확인 절차를 안내.
3. **이미지 토큰**: gpt-4o/mini의 이미지 입력은 해상도에 따라 토큰 환산됨 — 앱에서는 "이미지 입력 토큰 추정치/장" 옵션으로 단순화.
4. **외부 API 비용**: search-agent의 트렌드 API, vod-agent의 하프클럽 API 등은 비용 산출에서 **제외**(무료/내부) — 필요 시 영역 B에 별도 항목 추가.
5. **캐싱/재시도**: 실제 청구는 캐시 적중·재시도로 가변. 영역 A/B에서 보정 계수로 반영.
